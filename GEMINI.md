# Proyecto AI-Project: Mandatos y Estándares

Este archivo define las reglas de comportamiento y los estándares técnicos para este espacio de trabajo. Estas instrucciones tienen precedencia sobre los comportamientos por defecto.

## 🎯 Principios de Operación

1. **Precisión Absoluta:** La precisión es la prioridad número uno. Si no estás seguro de una solución o careces de información suficiente para resolver un problema de forma segura, **debes decir claramente que no lo sabes**. Nunca inventes soluciones o proporciones código especulativo.
2. **Streaming:** Se prefiere el streaming de respuestas para visualizar el proceso de razonamiento y generación de código en tiempo real siempre que la interfaz lo permita.
3. **Validación Empírica:** Antes de dar por finalizada una tarea, verifica que el código sea funcional y cumpla con los estándares definidos.

## 💻 Estándares Técnicos

### Desarrollo Web
- **Frontend:** React (TypeScript) por defecto.
- **Estilos:** Priorizar **Vanilla CSS** para mantener el proyecto ligero y flexible, evitando frameworks como Tailwind a menos que se solicite explícitamente.
- **Backend:** Node.js (Express) o FastAPI (Python) dependiendo de la complejidad.

### Automatización
- **Lenguajes:** Python o Node.js.
- **Simplicidad:** Preferir herramientas nativas y scripts directos sobre dependencias pesadas para tareas sencillas.

## 🛠️ Flujo de Trabajo

1. **Investigación:** Siempre analiza la estructura existente antes de proponer cambios.
2. **Explicación:** Describe brevemente la estrategia antes de ejecutar comandos que modifiquen el sistema.
3. **Tests:** Incluye scripts de prueba o instrucciones claras de validación para cada nueva herramienta o funcionalidad.
